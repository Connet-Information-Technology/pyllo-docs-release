Parameters#

Name/Tuple path of the option to set its value.

value: BasicTypes

Value to set on the configuration system.

section: Optional[str]

Section in the configuration system, e.g. shortcuts. If None, then the value of CONF_SECTION is used.

recursiveNotification: bool

If True, all objects that observe all changes on the configuration section and objects that observe partial tuple paths are notified. For example if the option opt of section sec changes, then the observers for section sec are notified. Likewise, if the option (a, b, c) changes, then observers for (a, b, c), (a, b) and a are notified as well.

New in version 1.0.0.

class pyllo.core.api.IConfigurationMixin#

Bases: IConfigurationObserver

This mixin inherits IConfigurationObservers feature and IConfigurationAccessors feature. Then enhance with 1. CONF_NAME_MAP:

Which split configuration into multiple .ini files

  1. CONF_DEFAULTS:

    Which adds additional defaults to pyllo.config.main.DEFAULTS at plugin integrating phase.

  2. CONF_FILE:

    Instruct config manager to read from and write to separate init files under sub folder named {CONF_SECTION} in pyllo configuration folder.

  3. CONF_VERSION:

    Configuration version.

Attention

Requires target object defines CONF_SECTION CONF_FILE, CONF_DEFAULTS, CONF_NAME_MAP, CONF_VERSION

New in version 1.0.0.

class pyllo.core.api.IPluginObserver#

Bases: object

This mixin interface enables a class to receive notifications when a plugin is available, by registering methods using the onPluginAvailable() decorator, and receive notification when a plugin is teardown, by registering methods using the onPluginTearDown() decorator.

When any of the requested plugins is ready, the corresponding registered method is called.

Attention

This mixin will only operate over the plugin requirements listed under REQUIRES and OPTIONAL class constants.

New in version 1.0.0.

REQUIRES: List[pyllo.core.dtype.PluginName] = []#

List of required plugin dependencies. Example: [Plugins.Plots, Plugins.IPythonConsole, ]. These values are defined in the Plugins class present in this file. If a plugin is using a widget from another plugin, that other must be declared as a required dependency.

New in version 1.0.0.

OPTIONAL: List[pyllo.core.dtype.PluginName] = []#

List of optional plugin dependencies. Example: [Plugins.Plots, Plugins.IPythonConsole, ]. These values are defined in the Plugins class present in this file. A plugin might be performing actions when connecting to other plugins, but the main functionality of the plugin does not depend on other plugins. For example, the Help plugin might render information from the Editor or from the Console or from another source, but it does not depend on either of those plugins. Methods in the plugin that make use of optional plugins must check existence before using those methods or applying signal connections.

New in version 1.0.0.
